# User Requirements Specification

URS-001: Researchers can select oncology context, targets, receptors, and candidate molecules.

URS-002: The system separates computational hypotheses from experimental evidence and blocks therapeutic claims.

URS-003: The system generates candidate-specific assay plans covering biochemical potency, target engagement, cellular response, selectivity, ADME, and early toxicity.

URS-004: The system exports lab-ready packets in JSON, CSV, Benchling-style CSV, SDF manifest, MOL2 manifest, and PDBQT manifest formats.

URS-005: Researchers can import assay results and receive a closed-loop recalibration summary.

URS-006: Wet-lab promotion requires a reason and two-person review.

URS-007: Candidate reports can be electronically signed and frozen.

URS-008: All workflow actions create exportable audit trail entries.

URS-009: Model outputs show uncertainty, applicability-domain status, evidence tier, and limitations.

URS-010: Production startup rejects placeholder secrets and non-production database settings.
