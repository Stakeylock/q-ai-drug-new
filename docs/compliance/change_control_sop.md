# Change Control SOP

1. Record proposed change, owner, reason, affected modules, risk class, and rollback plan.
2. Assess impact on data integrity, model behavior, audit logs, exports, and claim boundaries.
3. Update requirements, traceability, tests, model cards, and user-facing limitations.
4. Execute automated tests and focused manual workflow verification.
5. Approve with two-person review for high-risk scientific or compliance changes.
6. Freeze release notes and retain audit evidence.
