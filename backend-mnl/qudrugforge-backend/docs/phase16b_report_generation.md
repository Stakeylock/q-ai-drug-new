# Phase 16B Report Generation

Phase 16B generates downloadable report artifacts from persisted QuDrugForge backend data. Generated bytes are written through the storage abstraction and MongoDB stores only report metadata and file metadata.

## Routes

- `POST /api/v1/projects/{project_id}/reports/{report_id}/generate`
- `POST /api/v1/projects/{project_id}/reports/{report_id}/queue-generation`
- `POST /api/v1/projects/{project_id}/reports/generate-project-summary`
- `POST /api/v1/projects/{project_id}/reports/generate-candidate-dossier`

`queue-generation` is metadata-only for now: it sets a draft or failed report to `queued`. Actual rendering is synchronous in the `generate` endpoints.

## Generated Formats

- `csv`: `candidate_summary.csv`
- `sdf`: `candidates.sdf`
- `html`: `report.html`
- `pdf`: `report.pdf`

PDF rendering uses `reportlab`. RDKit is optional for SDF; when RDKit is absent or cannot parse structures, the service writes a metadata-only SDF fallback.

## Storage Paths

Generated files are saved under:

```text
storage/reports/{workspace_id}/{project_id}/{report_id}/{file_id}_{filename}
```

Examples:

```text
storage/reports/{workspace_id}/{project_id}/{report_id}/{file_id}_candidate_summary.csv
storage/reports/{workspace_id}/{project_id}/{report_id}/{file_id}_candidates.sdf
storage/reports/{workspace_id}/{project_id}/{report_id}/{file_id}_report.html
storage/reports/{workspace_id}/{project_id}/{report_id}/{file_id}_report.pdf
```

## File Metadata

Each generated file creates a `files` collection record with:

- `project_id`
- `workspace_id`
- `uploaded_by`
- `generated_by`
- `original_filename`
- `stored_filename`
- `file_type: generated_report`
- `mime_type`
- `local_path`
- `size_bytes`
- `checksum`
- `source_module: reports`
- `kind: generated`
- `artifact_type: csv | sdf | html | pdf`
- `linked_experiment_id`
- `metadata.report_id`
- `metadata.report_type`
- `metadata.format`
- `metadata.sdf_generation_method` for SDF files when relevant

The report document is updated with appended `file_ids`, `primary_file_id`, `completed_at`, and generation warnings in `metadata.warnings`.

## SDF Behavior

If candidate SMILES are available, SDF generation attempts RDKit first:

- `sdf_generation_method: rdkit` when RDKit writes structural records.
- `sdf_generation_method: metadata_only_fallback` when RDKit is unavailable or fails.

The fallback writes valid record separators and property blocks, including SMILES and candidate metrics, but the structure block is a zero-atom placeholder. If no candidate SMILES exist, SDF is skipped and the response includes a warning.

## HTML/PDF Contents

HTML and PDF include:

- title and generation timestamp
- project overview
- disease, cancer, target gene, and UniProt fields when available
- input and section completeness summaries
- candidate ranking table limited by `top_n`
- top candidate recommendation summary
- docking, GNINA, quantum/QML, ADMET, simulations, and artifact summaries
- generic wet-lab validation recommendations
- computational decision-support disclaimer

## Limitations

- Generation is synchronous in Phase 16B.
- No scientific compute is launched.
- Missing scientific results are not fabricated.
- Existing imported q-ai-drug report artifacts are not overwritten.
- Older generated report files are preserved and new file IDs are appended.
- Candidate ranking uses best available fields and falls back gracefully when fields are missing.

## Curl Examples

Generate an existing draft report:

```bash
curl -X POST "$API/api/v1/projects/$PROJECT_ID/reports/$REPORT_ID/generate" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "formats": ["csv", "sdf", "html", "pdf"],
    "include_sections": ["overview", "candidates", "docking", "gnina", "quantum", "admet", "simulations", "artifacts"],
    "top_n": 50
  }'
```

Create and generate a project summary:

```bash
curl -X POST "$API/api/v1/projects/$PROJECT_ID/reports/generate-project-summary" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Project Summary Report",
    "formats": ["pdf", "html", "csv"],
    "top_n": 50
  }'
```

Create and generate a candidate dossier:

```bash
curl -X POST "$API/api/v1/projects/$PROJECT_ID/reports/generate-candidate-dossier" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Candidate Dossier",
    "candidate_molecule_ids": [],
    "formats": ["pdf", "html", "csv", "sdf"],
    "top_n": 50
  }'
```

Download a generated file:

```bash
curl -L "$API/api/v1/files/$FILE_ID/download" \
  -H "Authorization: Bearer $TOKEN" \
  -o report.pdf
```
